입력 양식의 틀을 의미
form 태그 내의 submit 버튼을 클릭 시 해당 form 내에 작성된
사용자가 입력한 값들을 서버로 넘기면서 요청하는 역할을 수행

[ 표현법 ]
form action="" method=""
* action 속성 : 해당 form 내에 사용자가 입력한 값들을 전달할 서버의 경로를 제시
* method 속성 : 요청 전송 방식을 지정하는 속성 (get / post)
> get 방식 : 요청 시 사용자가 입력한 값들이 url 에 노출되는 방식 (생략 시 기본값)
> post 방식 : 요청 시 사용자가 입력한 값들이 url 에 노출되지 않는 방식
(post 방식 같은 경우 주로 민감정보를 넘길 때 사용함 => 로그인, 회원가입, ..)

form 태그 내의 제출(submit) 버튼 클릭 시 action 에 지정한 서버로 method 에 작성한 요청 전송 방식으로 사용자가 입력한 값들을 key=value 세트로 전달
=> key=value&key=value&.. 를 쿼리스트링 이라고 부른다.

라디오 버튼과 체크박스 관련 input 태그의 type

라디오 버튼 (type="radio")

- 제시한 여러개의 값들 중에서 오로지 한개만 선택해야 할 때
(name 속성값이 똑같은 것들끼리 하나의 그룹으로 이어짐)
- 제출 시 현재 선택된 (checked) 값 하나만 넘어감

성별 :

체크박스 (type="checkbox")

- 제시한 여러개의 값들 중에서 여러개를 선택 가능한 경우
- 현재 선택된 (checked) 값들이 넘어감
- 넘기고자 하는 value 값이 넘어가고, 이 때 배열 형식으로 넘어가게 됨

select 태그와 option 태그

- 사용자에게 직접 키보드로 입력받는게 아니라 내가 제시한 여러 옵션들 중에서 선택할 수 있게끔 해주는 태그
- 제출 시 현재 선택된 (selected) 값이 넘어감
- select 로 옵션들을 묶어주고 option 으로 여러 선택지를 제시하면 됨
- 각각의 option 에 value 속성값을 명시하면 해당 value 값이 전달됨
- 만일, value 속성을 명시하지 않았을 경우 기본적으로 option 태그의 content 영역 (시작태그와 종료태그 사이의 문구) 값이 넘어감
